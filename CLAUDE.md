# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

SpotifAI is an AI-powered Spotify playlist generator. An LLM agent orchestrated via LangGraph interacts with the Spotify Web API through natural language.

## Development Commands

```bash
# Install in editable mode with dev dependencies
uv pip install -e .[dev]

# Run the application
uv run spotifai
# or
python -m spotifai

# Format code
uv run black src
uv run isort src

# Run tests
uv run pytest -q
```

## Environment Setup

Create a `.env` file with:
```
OPENAI_API_KEY=...
DEFAULT_MODEL_NAME=gpt-4o-mini
DEFAULT_MODEL_PROVIDER=openai
```

Spotify OAuth tokens are cached at `~/.spotifai/.cache`. Self-signed SSL certificates for the local OAuth callback server are generated at `~/.spotifai/cert.pem` and `~/.spotifai/key.pem` on first run.

## Architecture

### LangGraph State Machine (`src/deepagent/agent.py`)

The core is a `DeepAgent` that builds a LangGraph state machine with the following node flow:

```
planner → researcher → [summarizer →] executor ⇄ tools
                                         ↓
                                       critic → [researcher (loop)] → finalizer
```

- **planner**: Decomposes user query into ordered steps (outputs `Plan`)
- **researcher**: Analyzes the current plan step to produce an `Intent` (goal + notes)
- **summarizer**: Compresses `messages` history when count > 10 (uses a sentinel to mark truncation boundary)
- **executor**: Decides next action — calls tools, asks user, or signals finalization
- **tools**: Executes LangChain tools (or mocked tools for testing via `__tools_mock`)
- **critic**: Evaluates progress; loops back to researcher if more steps remain
- **finalizer**: Generates the final user-facing response

State is defined in `src/deepagent/state.py` with a custom `messages` merge function that handles the sentinel-based history compression.

### Spotify Integration (`src/spotify/`)

- `spotify_manager.py` — Spotipy wrapper for all Spotify API operations (search, create/modify playlists, manage tracks). Handles pagination automatically.
- `spotify_callback.py` — Local HTTPS server (FastAPI + uvicorn with self-signed cert) that handles the OAuth2 PKCE redirect.
- `spotify_tools.py` — LangChain `@tool`-decorated wrappers around `SpotifyManager` methods; these are passed to `DeepAgent`.

### Prompt System (`src/deepagent/prompts.py` + `src/deepagent/assets/`)

Prompts are Markdown templates loaded from `assets/` and rendered with variables. The `Prompts` class exposes `get(name)` and `render(name, variables)`. Tool signatures are auto-generated by `src/deepagent/tools.py` from the `@tool` decorators and injected into `INSTRUCTIONS.md`.

### Streaming Output (`src/deepagent/chunk.py`, `src/deepagent/chat.py`)

`DeepAgent.invoke()` is an async generator that yields `Chunk` objects (`ChunkType.TEXT` or `ChunkType.THINKING`). `chat()` in `chat.py` drives the interactive REPL loop.

## Key Design Decisions

- The agent uses **mocked tools** (`__tools_mock`) for dry-run/testing — this avoids real Spotify API calls during development.
- The **summarizer** node only activates when `len(messages) > 10`, controlled by the `__need_summarize` edge condition.
- The **critic** loop between executor→tools→executor can repeat many times per plan step; the outer critic→researcher loop advances to the next plan step.
- Spotify OAuth uses **PKCE** (not client secret) — the `SPOTIFY_CLIENT_ID` is public and embedded in `src/spotify/__init__.py`.
